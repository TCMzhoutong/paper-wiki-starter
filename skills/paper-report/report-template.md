# 通用骨架

本文件由 SKILL.md 阶段 3 触发加载。

## 文件命名与位置

```
wiki/reports/<主题简称>_<YYYY-MM-DD>.md
```

主题简称约束：
- 4-12 个汉字或英文词
- 避免冒号、斜杠、空格、引号
- 同主题不同时间多次跑 → 不同日期后缀，旧 report 不主动归档

## Frontmatter

严格按 `paper_wiki/schema/prompts/wikilink-format.md` 的"报告"章节：

```yaml
---
title: "完整报告主题"
date: YYYY-MM-DD
tags: [领域标签1, 领域标签2]
papers: ["[[作者1_年份_关键词]]", "[[作者2_年份_关键词]]"]
related: ["[[相关概念]]", "[[相关报告]]"]
---
```

⚠️ 所有 wikilink 必须用双引号包裹（YAML 数组里裸 `[[X]]` 会被解析为嵌套列表，导致 Grep 失败）。

## 正文骨架

```markdown
# {{完整报告主题}}

## 问题

{{用户的原始问题，原话或最小改写。不要替用户重写问题}}

## 结论

{{1-3 句核心结论。关键术语用 [[双链]]。结论必须是"答"，不是"我们将分析"}}

## 分析

{{此处是积木装配区。

填充规则：
  - 每个积木块 = 一个 ### 子节（h3，不是 h2）
  - 子节标题由积木 micro-template 决定（matrix 用维度族名 / timeline 用主题名 / gap-ranking 固定为"缺口总览"）
  - 多个积木顺序：陈列型在前（提供事实），提炼型在后（提供洞察）}}

## 涉及论文

- [[作者1_年份_关键词]] — 在本报告中的角色（一句话，例如 "评估深度最完整的样本"）
- [[作者2_年份_关键词]] — 在本报告中的角色

## 积木使用记录

> [!note] Skill 元信息（用户审计用）
> - 意图：{{GAP_FINDING / LANDSCAPE / ... / OTHER}}
> - 积木：{{matrix, gap-ranking, ...}}
> - 替代/缺失：{{无 / 原本应用 X，因缺失改用 Y}}
> - 路径：{{Claude 直读 / NotebookLM}}
> - 论文确认闸门：{{用户已确认 / 跳过（违规）}}
```

## 写完后联动规则

- **report 不反向更新概念页** —— report 是查询快照，不是概念的永久属性
- **不主动创建论文/概念 stub** —— 引用的对象若不存在，仍写 `[[X]]` 但不建文件，避免污染 wiki
- **跨多份 report 反复出现的概念但 wiki/concepts/ 缺页** → 在元信息段额外加一行 "建议建立概念页：[[X]]"，让用户决定是否手动建

## 反例（不要这样写）

❌ 结论：本报告将从评估数据集、自动化指标、专家评估三个维度对比 X、Y、Z 三篇论文
（这是导言，不是结论；结论应该回答用户问题）

❌ 分析下没有任何子节，全是流水段落
（违反积木式装配，丢失结构）

❌ Frontmatter 写 `papers: [[[作者_年份]]]`
（缺双引号，YAML 解析会出错）
