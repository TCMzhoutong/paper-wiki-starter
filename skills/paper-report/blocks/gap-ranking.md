# 积木：缺口识别 + 排序

## 适用

GAP_FINDING 的核心积木，也可作为 LANDSCAPE / METHOD_COMPARE 的收尾提炼。

⚠️ **必须**与某个陈列型积木（matrix / timeline / ...）配合使用——gap 是从陈列中浮现的，不是凭空提的。

## micro-prompt

1. 回看本 report 中的陈列型积木（matrix 的"分歧""缺失" / timeline 的"留白"）
2. 从中提炼 2-4 个 gap，每个 gap 必须满足：
   - **可证伪**：有具体场景 / 任务，不是"做得还不够好"这种空话
   - **有依据**：能指向陈列积木中的具体单元格作为出处
3. 为每个 gap 评估：
   - **难度**（低 / 中 / 高）：实现需要的资源与技术挑战
   - **价值**（⭐~⭐⭐⭐）：解决后的科研 / 工程影响
   - **可借鉴对象**：现有论文或邻域工作中可作起点的方案
4. 按"价值降序、难度升序"排序

## 约束

- 单个 gap 描述 **≤ 30 字**，便于扫读
- "可借鉴对象"必须是 report 涉及论文集合内的某篇，或显式标注"邻域外参考：..."
- 不要凑数：宁可只列 2 个真 gap，不要列 4 个掺水的
- "依据出处"列必须能让读者反查到陈列积木中的具体位置

## micro-template

```markdown
### 缺口总览

| 缺口 | 难度 | 价值 | 可借鉴对象 | 依据出处 |
|---|---|---|---|---|
| {{≤30 字描述}} | 低 / 中 / 高 | ⭐~⭐⭐⭐ | [[作者_年份_关键词]] | 见上文"{{矩阵子节名}}"的{{某行某列 / 某观察项}} |
| ... | ... | ... | ... | ... |

**最有开题价值的 1 个**：{{挑出价值最高的一个，1-2 句说明为什么值得优先开题}}
```

## 反例

❌ Gap 写"中医 LLM 还需要更多研究" → 太空，不可证伪
❌ 依据出处列空着 → 必须能反查到陈列积木中的具体单元格
❌ 列出 5 个 gap 但其中 2 个明显是凑数 → 删减到 2-3 个真 gap
❌ "可借鉴对象"写"现有 RAG 工作" → 必须具体到 [[作者_年份_关键词]]
