# 积木：跨论文对比矩阵

## 适用

当问题需要在多个维度上横向比较多篇论文时。陈列型积木的默认选择。

## micro-prompt

1. 从用户问题中识别 N 个对比维度（N=3-6，太少信息量不足，太多变流水账）
2. 维度可以是：方法、数据集、指标、性能数值、限制、特定设计选择、超参数
3. 为每个 (论文 × 维度) 单元格填入 **具体内容**（数值、方法名、设计名），而不是模糊形容
4. 矩阵下方写"观察"段：
   - **共性** 1-2 条（多篇论文都做了什么）
   - **分歧** 1-2 条（哪些维度上方法分裂）
   - **缺失** 0-1 条（哪些维度上多数论文没做，可与 gap-ranking 衔接）

## 约束

- 一个 matrix 积木 = 一个 ### 子节
- 如果对比维度可分组（如"评估指标族 vs 数据集族 vs 成本族"），用多个 matrix 积木分别陈列，每个一个子节
- 数值要保留具体单位和小数位，不要简化为"约 70%"
- 若某单元格论文中没提及 → 写 `❌` 或 `未报告`，不要捏造或写"未知"

## micro-template

```markdown
### {{维度族名，如 "数据集" / "评估指标" / "方法架构" / "训练设置"}}

| 论文 | {{维度1}} | {{维度2}} | {{维度3}} | ... |
|---|---|---|---|---|
| [[作者1_年份_关键词]] | ... | ... | ... | ... |
| [[作者2_年份_关键词]] | ... | ... | ... | ... |
| [[作者3_年份_关键词]] | ... | ... | ... | ... |

**观察**：
- 共性：...
- 分歧：...
- 缺失（可选）：...
```

## 反例（不要这样填）

❌ 单元格写"使用了大模型" → 应写 "GPT-4 + text-embedding-ada-002"
❌ 单元格写"性能较好" → 应写 "MLE 准确率 75.67% (95% CI 68.97-80.68)"
❌ 观察段写"三篇论文都很优秀" → 应写具体共性，如"都使用了 GPT-4 作为基座生成模型"
