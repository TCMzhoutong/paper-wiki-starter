# 积木库索引

每个积木 = 一个 micro-template（markdown 结构）+ 一个 micro-prompt（如何填）。

新增积木的成本：写一个 `blocks/<name>.md`（约 30-50 行）+ 在本文件登记一行。

## 陈列型（必选至少 1 个）

| 积木 | 文件 | 适用 | 状态 |
|---|---|---|---|
| 对比矩阵 | matrix.md | 多论文 × 多维度横切 | ✅ v0.1 |
| 时间线 | timeline.md | 时间维度上的演进 / 出现序列 | ✅ v0.1 |
| 资源对比表 | resource-table.md | 数据集 / 工具 / baseline 横切 | ⏳ v0.x |
| 分类树 | taxonomy.md | 工作的类别归纳与代表 | ⏳ v0.x |

## 提炼型（可选 1-2 个）

| 积木 | 文件 | 适用 | 状态 |
|---|---|---|---|
| 缺口识别+排序 | gap-ranking.md | 从对比中提炼 gap，按难度×价值排序 | ✅ v0.1 |
| Trade-off 分析 | tradeoff.md | 多方案的优劣权衡 + 选型/联用建议 | ✅ v0.2 |
| 因果归因 | attribution.md | Push/Pull/结果三段式拆解演进驱动力 | ✅ v0.3 |

## 当前缺失积木的临时替代规则

| 缺失积木 | 临时替代 |
|---|---|
| resource-table | matrix（行=资源，列=对比维度），标注 |
| taxonomy | matrix（行=类别，列=代表工作），标注 |

**所有替代必须在 report 末尾"积木使用记录"段落显式记录**——这是积木库扩充的真实需求来源。静默吞掉等于让 skill 永远停在当前缺失状态。

## 积木升级路径

观察到某种替代连续出现 ≥3 次 → 该积木需求被验证 → 优先新建。
新建步骤：
1. 收集 3 次替代的实际填充内容，提炼共同结构
2. 写 `blocks/<name>.md`（参考 matrix.md 的格式）
3. 更新本索引 + meta-prompt.md 中的"推荐组合"表
4. 缺失替代规则中删除该项
