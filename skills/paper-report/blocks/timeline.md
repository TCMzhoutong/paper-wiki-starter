# 积木：时间线

## 适用

EVOLUTION / ATTRIBUTION / LANDSCAPE 类问题。展示技术、概念、方法在时间维度上的出现序列与演变。

## micro-prompt

1. 从论文集合中按时间排序（精确到月，参考 frontmatter `date`）
2. 每个时间节点标注：
   - 论文（带 wikilink）
   - 动作 / 贡献（一句话）
   - 触发因素（若论文中有提及；不明示则标 `(推测)` 或留空）
3. 时间线下方写"演进观察"段：
   - **主线** 1-2 条（什么是延续性发展）
   - **转折** 0-2 条（什么时候出现范式转移）
   - **留白** 0-1 条（哪些时段没有动作，可能预示空白）

## 约束

- **论文跨度 < 2 年时，时间线意义有限，应改用 matrix**
- 时间标注精确到月（YYYY-MM），避免"近年来"、"早期"这种模糊词
- 触发因素若不在论文中明示，标注 `(推测)` 或不写，不要把"演进观察"中的归因塞进单元格
- 同月多篇论文 → 同一行下分多条，按字母序

## micro-template

```markdown
### 时间线：{{主题}}

| 时间 | 论文 | 动作 / 贡献 | 触发因素 |
|---|---|---|---|
| YYYY-MM | [[作者_年份_关键词]] | ... | ... |
| YYYY-MM | [[作者_年份_关键词]] | ... | ... |

**演进观察**：
- 主线：...
- 转折（若有）：...
- 留白（若有）：...
```

## 反例

❌ 时间列写"早期" / "近期" → 应写具体 YYYY-MM
❌ 触发因素列凭脑补填写"因为 GPT-4 出现了" → 论文未明示则留空或标 `(推测)`
❌ 三篇论文都在同年同月发表却用 timeline → 改用 matrix
