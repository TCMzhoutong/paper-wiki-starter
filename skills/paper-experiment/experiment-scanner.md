# Experiment 可抄素材信号扫描规则

本文件由 paper-experiment skill 阶段 3 触发加载。

## 信号类别（6 种）

从刚生成的 paper-report 扫以下"可直接照抄"的素材信号，区别于 idea-scanner 扫的"gap 信号"：

| # | 信号类别 | 在 report 中的位置 | 信号强度 |
|---|---|---|---|
| 1 | **matrix 单元格的具体 baseline / 方法配置** | matrix 表格"方法/机制/配置"列的非 ❌ 单元格 | ⭐⭐⭐（可直接抄） |
| 2 | **matrix 单元格的 dataset / cohort / 样本** | matrix 表格"数据集/样本/队列"列的非 ❌ 单元格 | ⭐⭐⭐ |
| 3 | **matrix 单元格的 metric / 终点 / 检测指标** | matrix 表格"评估方式/指标"列的非 ❌ 单元格 | ⭐⭐⭐ |
| 4 | **tradeoff 某路径的"优势"条目** | 某路径优势段（含具体数值/配置） | ⭐⭐ |
| 5 | **tradeoff 某路径的"劣势"条目**（踩坑警示） | 某路径劣势段 | ⭐⭐（反向有用） |
| 6 | **开源 / 注册信息关键词** | 扫关键词：`github` / `available at` / `open-source` / `ChiCTR` / `NCT` / `repo` | ⭐（指向 repo / 注册号） |

## 扫描算法

1. Read paper-experiment 阶段 2 生成的 report 文件
2. 按 6 类信号顺序扫，每命中一条生成一个候选
3. 候选上限 **10 个**，超过按信号强度降序保留前 10
4. 同一 report 段落只产一个候选——如 matrix 某行的 baseline + 配置都选中，合并为一条（因为"抄 X 的方法"本身就含超参）

## 每个候选的字段

```
| # | 一句话概要 (≤30 字) | 信号类别 | 信号出处 | effort | risk |
```

- **一句话概要**：**动词开头**，如"抄 Tang 的 BERT-BiLSTM-CRF NER 配置"、"用 Zhu 的贝叶斯舌脉关联规则"
- **信号出处**：具体到 matrix 行列或 tradeoff 段，如"模态与机制表 Tang 行 + 融合机制列"
- **effort 粗估**：
  - `低`：配置简单可直接 copy-paste（有开源代码）
  - `中`：需适配某些依赖 / 改 fork
  - `高`：仅有论文描述，需重新实现 / 需做新伦理申请
- **risk 粗估**：
  - `低`：有公开 repo / SOP 可查 / 注册号可核实
  - `中`：仅论文描述，需自行推断
  - `高`：作者自述存在复现障碍 / 数据未公开

⚠️ effort / risk 粗估允许 LLM 判断，但阶段 5.2 写卡时会被继承到 frontmatter，用户在阶段 4 可质疑 / 修改。

## 呈现格式（给用户看的）

```markdown
## 本 report 浮现的 experiment 候选

| # | 一句话概要 | 信号类别 | 出处 | effort | risk |
|---|---|---|---|---|---|
| 1 | 抄 Tang 的 CNN+RNN+注意力融合 | ⭐⭐⭐ matrix-配置 | 模态与机制表 Tang 行 | 中 | 低 |
| 2 | 用 Zhu 的贝叶斯舌脉关联做决策级 | ⭐⭐⭐ matrix-配置 | 模态与机制表 Zhu 行 | 中 | 中 |
| 3 | 复用 Wu 综述里 TCM-FTP 的 LoRA + 排列增强 | ⭐⭐ tradeoff-优势 | 路径 C 优势 2 | 低 | 低 |
| ... |

请回复要转为 experiment 卡的编号（如 "1 3"）。多选时：
- 默认自动判断合并或拆分（同 `experiment_type` 且共享 baseline → 合并；否则拆）
- 显式指定：`1+3 合并` 强制合一张；`1,3 分开` 强制拆两张
其他：`全要` / `不要` / `换一组`
```

## 扫不到信号时

候选数 = 0 时，不写空卡，回报：

> 本 report 在 6 类信号下未发现可直接照抄的素材。
> 可能原因：
> 1. report 主题过于理论/综述，缺乏具体实验对象
> 2. 积木组合未包含 matrix（仅靠 gap-ranking 无法扫到执行层素材）
> 3. 涉及论文多为方法论文，未报告完整配置
>
> 建议：
> - 换题到更具体的可操作方向
> - 或要求 paper-report 必选 matrix 积木（override 强制）
> - 或补充精读更多实证论文后再跑

## 扫描的保守原则

- **宁可少不要多**：5 条强候选 > 10 条掺水候选
- **宁可不提不要凭空造**：每条候选必须有信号出处可追溯
- ⭐ 单星信号（类别 6）要额外谨慎：外部 repo / 注册号可能已失效，优先列，但在卡片里阶段 5.1 必须用 qmd 核实或手工核实后再入卡
