# Experiment 卡片模板

本文件由 paper-experiment skill 阶段 5.2 触发加载。

## 文件命名

```
wiki/experiments/<短标题>_<YYYY-MM-DD>.md
```

**短标题约束**：4-12 个汉字或英文词；避免冒号、斜杠、空格、引号。

## Frontmatter

```yaml
---
title: "实验方案完整标题"
date: YYYY-MM-DD
status: planned              # planned / running / done / abandoned
experiment_type: computational   # computational / animal / clinical / in-vitro / omics / mixed / other
tags: [领域标签]
source_report: "[[支撑 report 文件名]]"
source_idea: null            # 或 "[[衍生自的 idea 卡文件名]]"
target: "一句话：要做什么实验"
effort: low                  # low / medium / high
risk: low                    # low / medium / high
related_papers: ["[[提供 baseline 的论文]]"]
related_concepts: ["[[核心概念]]"]
---
```

⚠️ 所有 wikilink 必须用双引号包裹。

`experiment_type: other` 时必须补一句话描述，格式：`experiment_type: "other: <一句话描述>"`

## Status 流转规则

| Status | 语义 | 触发条件 | 末尾要求 |
|---|---|---|---|
| `planned` | 方案写完未启动 | skill 首次生成默认 | — |
| `running` | 实验进行中 | 用户手动改 | 加"进度日志"段（按日期追加） |
| `done` | 实验完成 | 用户手动改 | 链接产出论文 / 实验 report；填"与原方案差异记录" |
| `abandoned` | 放弃 | 用户手动改 | 补"放弃原因"段 |

## 类型变体：第 3 段 "研究对象 + 具体配置" 子模板

### computational（工科 AI / ML）

```markdown
- **数据集**：{{名称 + 规模 + 许可证 + 下载源}}
- **预处理**：{{步骤 + 脚本}}
- **划分**：train / val / test = X / Y / Z
- **模型配置**：
  - 基座：{{Qwen2.5-7B / Llama 3 / ...}}
  - 微调方式：{{LoRA r=X, lr=Y, batch=Z, epochs=N, warmup=W}}
  - 损失函数：{{CE / LMERL / ...}}
- **硬件**：{{N × GPU 型号 + 显存 + 预计时长}}
- **参考 repo**：{{github link / 需自实现}}
```

### animal（动物实验）

```markdown
- **物种 / 品系**：{{SPF 级 C57BL/6J 小鼠 | Wistar 大鼠 | ...}}
- **年龄 / 性别 / 体重**：{{雄性 6-8 周龄 / 18-22g}}
- **动物来源**：{{动物中心 + 许可证号}}
- **样本量**：n = X（每组 Y 只 × Z 组），依据 {{G*Power / 效应量计算过程}}
- **伦理审批**：{{机构动物伦理委员会批号}}
- **造模方法**：{{模型类型 + 诱导方式 + 成模判定}}
- **受试物**：{{方剂 / 提取物 / 组分 + 制备方法 + 质控}}
- **剂量组**：低 X mg/kg / 中 Y / 高 Z + 阳性对照 + 模型对照 + 空白对照
- **给药**：{{ig/ip/po}}，每日 N 次 × M 天
- **取材**：{{时间点 + 组织部位 + 保存方式}}
```

### clinical（人体临床试验）

```markdown
- **疾病类型 + 诊断标准**：{{ICD / 中医辨证 / 专病标准}}
- **纳入标准**（3-5 条）：
  1. ...
- **排除标准**（3-5 条）：
  1. ...
- **样本量**：n = X，基于主要终点效应量 + α = 0.05 + power = 0.8 估算
- **试验类型**：RCT / 队列 / 病例对照 / 单臂 / 交叉 / 真实世界
- **随机化**：{{方法 + 区组 + 分层因素}}
- **盲法**：双盲 / 单盲 / 开放
- **注册号**：{{ChiCTR-XXX / NCT-XXX}}
- **伦理审批**：{{机构伦理委员会批号}}
- **招募来源**：{{单 / 多中心列表}}
- **干预方案**：
  - 试验组：{{方药 / 针灸穴位 / 手法}}
  - 对照组：{{安慰剂 / 阳性药 / 标准治疗}}
- **随访周期**：{{时间点}}
- **依从性监测**：{{方法}}
```

### in-vitro（体外细胞 / 生化）

```markdown
- **细胞系 / 器官模型**：{{名称 + 来源（ATCC / CCTCC / 合作实验室）}}
- **鉴定**：{{STR / mycoplasma}}
- **培养条件**：{{培养基 + 血清 + 抗生素 + 温度 + CO2}}
- **分组设计**：对照 / 模型 / 给药（梯度剂量）/ 阳性药
- **受试物制备**：{{浓度梯度 + 溶剂 + 过滤 / 灭菌}}
- **处理时长**：{{N 小时 / 天}}
- **检测方法**：
  - 分子：{{Western blot / qPCR / 免疫荧光 / ELISA}}
  - 表型：{{MTT / CCK-8 / 流式 / Transwell / 划痕}}
  - 标靶：{{蛋白 / 基因 / 通路列表}}
- **重复**：技术重复 × N / 生物学重复 × M
```

### omics（多组学 / 生信）

```markdown
- **样本来源**：{{组织 / 细胞 / 血液 + 数量 + 来源 + 保存方法}}
- **测序 / 检测平台**：{{Illumina NovaSeq / Nanopore / Orbitrap / ...}}
- **测序深度 / 质谱参数**：{{reads per sample / resolution}}
- **公开数据集补充**：{{GEO GSE-X / TCGA 项目 / MetaboLights}}
- **分析 pipeline**：
  - QC：{{FastQC / Trimmomatic}}
  - 比对 / 定量：{{STAR / Salmon / MaxQuant}}
  - 差异分析：{{DESeq2 / limma / MetaboAnalyst}}
  - 富集：{{GO / KEGG / GSEA / Reactome}}
- **网络药理学**（若涉及）：{{TCMSP / HERB / ETCM + STRING / Cytoscape}}
- **硬件**：{{服务器 CPU / 内存 / 存储需求}}
```

### mixed（多类型组合）

mixed 类型的"研究对象 + 具体配置"段**内嵌多个子段**，按 pipeline 时间顺序：

```markdown
### 阶段 1：{{类型 + 目标}}

{{按对应类型子模板填所有字段}}

**本阶段产出（作为下阶段输入）**：{{...}}

### 阶段 2：{{类型 + 目标}}

{{按对应类型子模板填}}

**本阶段依赖上阶段的**：{{...}}
**本阶段产出**：{{...}}

### 阶段 3：...
```

⚠️ 各阶段之间**必须显式写衔接关系**（哪个阶段的什么产出作为下阶段输入），否则 mixed 的价值反而丢失。

### other（自定义）

```markdown
**自定义类型描述**：{{一句话说明是什么类型的实验（frontmatter 已标注，此处展开）}}

{{按类似"对象 + 配置"结构自拟字段，参考前面 6 种类型的字段组合}}
```

⚠️ other 使用率 ≥ 3 次后，应评估是否升级为标准类型（加入上面 6 类）。

## 字段用途速查

| 字段 | 回答什么问题 |
|---|---|
| `status` | experiment 当前生命周期阶段 |
| `experiment_type` | 实验类型（决定第 3 段用哪个变体） |
| `target` | 这实验最终要证明什么 |
| `source_idea` | 这 experiment 是哪个 idea 长出来的 |
| `effort` | 做一遍要多大代价 |
| `risk` | 会不会失败到无产出 |
| `目标实验` | 要做什么、成功标准 |
| `借鉴的先前研究` | 我抄谁的 baseline / 对照 |
| **`研究对象 + 具体配置`** | **实操每个参数填什么（experiment 的核心杠杆）** |
| `评估指标 / 终点 / 检测指标` | 怎么判断是否成功 |
| `实验设计` | 分组 / 对比 / 消融怎么摆 |
| `实施 checklist` | 今天就能开始跑的动作 |
| `已知陷阱` | 不想踩的坑 + 规避 |
| `与原方案差异记录` | 我改了什么、为什么 |
| `溯源` | 这方案从 report 哪里冒出来的 |

## 反例（不要这样写）

❌ "借鉴的先前研究"只写论文名，没写具体配置
　　→ 必须精确到超参 / 剂量 / 纳排标准

❌ `experiment_type: animal` 但"研究对象"段填数据集/超参
　　→ 类型决定必填字段；类型错了整张卡价值归零

❌ "实施 checklist" 写"开始实验" / "跑代码"
　　→ 不是原子级。应写"购买 C57BL/6J 小鼠 30 只并做 1 周适应性饲养"、"clone `git@github.com:...` 并跑 `python train.py --config base.yaml`"

❌ `status=done` 但"与原方案差异记录"段空
　　→ 实验跑完必有差异，done 必填

❌ mixed 类型各阶段写完但没写衔接关系
　　→ mixed 的核心价值就在衔接，不写等于写了三张独立卡硬拼在一起

❌ 配置 / 剂量 / 纳排标准没 qmd 出处
　　→ 违反 5.1 规则，等同 LLM 凭背景知识填，不可审计
