# 论文精读 — NotebookLM 提问模板

依次执行以下 notebooklm ask 命令。每条独立提问，结果保存为笔记。

> [!warning] 前后置约束（CLAUDE.md 核心约束 — NotebookLM 笔记必须闭环）
> **前置**：notebook 已通过 `notebooklm create "<title>"` 创建，并 `notebooklm use <id>` 设置为当前上下文（或后续 ask 命令显式带 `-n <id>`）
> **后置**：本文件所有 ask 命令执行完毕后，**必须**执行 `notebooklm delete -n <id> -y` 清理本次精读创建的 notebook
> 即使中途某个 ask 失败 / 用户中断，也要尝试 cleanup。失败时显式标注 notebook-id 让用户手动清，不可静默吞掉

## 提问序列

### 1. 结构化摘要

```bash
notebooklm ask "请对这篇论文进行结构化摘要，包含以下字段：
- 研究背景（2-3句）
- 研究目标
- 核心方法
- 数据集/实验设置
- 主要结果（含关键数值）
- 局限性
- 未来方向" --save-as-note --note-title "结构化摘要"
```

### 2. 方法细节

```bash
notebooklm ask "详细拆解这篇论文的技术方案：
- 整体框架/pipeline 是什么？
- 每个模块的输入、处理、输出是什么？
- 使用了哪些模型/算法？参数设置？
- 与 baseline 方法的关键区别是什么？
- 有哪些创新点？" --save-as-note --note-title "方法细节"
```

### 3. 实验与结果

```bash
notebooklm ask "总结这篇论文的实验设计和结果：
- 评估指标有哪些？
- 各方法的性能对比（列出具体数值）
- 消融实验的结论
- 最重要的发现是什么？
- 结果是否支持了作者的核心论点？" --save-as-note --note-title "实验与结果"
```

### 4. 关键概念提取

```bash
notebooklm ask "从这篇论文中提取所有关键概念，每个概念包含：
- 概念名称（中文规范名）
- 中英文对照（英文全称）
- 常见缩写或同义变体（如 LLM、GPT、CoT；没有则写'无'）
- 一句话定义
- 在本文中的角色（核心方法/背景知识/评估指标/数据集等）
- 与其他概念的关系

输出格式（每个概念一行）：
概念1: 中文名 | 英文: 英文全称 | 缩写: X, Y | 定义: ... | 角色: ... | 关联: 概念2, 概念3
概念2: 中文名 | 英文: 英文全称 | 缩写: 无 | 定义: ... | 角色: ... | 关联: 概念1

要求：
- 中文名作为规范名（优先选学界通用的中文表述）
- 英文和缩写用于 Obsidian aliases，须准确对应该概念的标准变体
- 不要强行翻译英文独有术语（如 Transformer 保持原名）" --save-as-note --note-title "关键概念"
```
